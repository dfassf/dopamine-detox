# Phase 2: 핵심 기능 — 금욕 타임라인 코칭

## 개요

로그인한 사용자가 금욕을 등록하고, 개인화된 신체 변화 타임라인을 받는 핵심 경험.
"눈에 안 보이는 변화를 보여준다"가 핵심 가치.

---

## 금욕 등록

### 금욕 종류

| 구분 | 종류 | 타임라인 데이터 |
|---|---|---|
| 프리셋 | 금주, 금연, 식단 | 의학 근거 기반 타임라인 제공 |
| 커스텀 | 사용자 직접 입력 (예: 카페인, SNS, 야식 등) | 범용 심리 타임라인 제공 |

- 한 사용자가 여러 금욕을 동시에 등록할 수 있다.
- 프리셋 종류는 추후 확장 가능.

### 등록 시 수집 정보

#### 공통 (모든 종류)

| 필드 | 타입 | 필수 | 용도 |
|---|---|---|---|
| 금욕 종류 | select / text | O | 프리셋 선택 또는 커스텀 입력 |
| 시작일 | date | O | D+N 계산 기준 |
| 생년 | number (년도) | O | 나이 → 회복 속도 반영 |
| 성별 | select | O | 호르몬, 체성분 차이 반영 |

#### 금주 추가

| 필드 | 타입 | 필수 | 용도 |
|---|---|---|---|
| 체중 | number (kg) | O | 대사량 기반 타임라인 |
| 키 | number (cm) | O | BMI 참고 |
| 음주 기간 | number (년) | O | 손상 정도 추정 |
| 음주 빈도 | select | O | 매일 / 주 3~5회 / 주 1~2회 |
| 음주량 | text | O | 예: "소주 2병" |

#### 금연 추가

| 필드 | 타입 | 필수 | 용도 |
|---|---|---|---|
| 흡연 기간 | number (년) | O | 폐 손상 정도 추정 |
| 일일 흡연량 | number (개비) | O | 니코틴 의존도 |

#### 식단 추가

| 필드 | 타입 | 필수 | 용도 |
|---|---|---|---|
| 체중 | number (kg) | O | 체성분 변화 추적 |
| 키 | number (cm) | O | BMI 참고 |
| 목표 | select | O | 다이어트 / 건강식 전환 / 기타 |

#### 커스텀 추가

| 필드 | 타입 | 필수 | 용도 |
|---|---|---|---|
| 대상 이름 | text | O | 예: "카페인", "야식" |
| 해당 습관 기간 | number (년) | X | 있으면 타임라인에 반영 |

> 공통 필드(생년, 성별)는 최초 등록 시 입력하면 이후 등록에서 자동 채움.

### 등록 플로우

```
[메인 화면]
    │
    ├─ "금욕 시작하기" 클릭
    │
    ▼
[종류 선택]
    │  프리셋 (금주/금연/식단) 또는 커스텀 입력
    │
    ▼
[상세 정보 입력]
    │  시작일 + 공통 정보 + 종류별 추가 정보
    │
    ├─ 등록 완료
    │
    ▼
[타임라인 화면으로 이동]
    │  "당신의 타임라인이 생성되었습니다"
```

### 금욕 등록 API

```
POST /api/abstinence

Request (금주 예시):
{
  "type": "alcohol",
  "start_date": "2026-01-26",
  "birth_year": 1989,
  "gender": "male",
  "weight": 87,
  "height": 175,
  "drinking_years": 5,
  "drinking_frequency": "daily",
  "drinking_amount": "소주 2~4병"
}

Response (201):
{
  "id": 1,
  "type": "alcohol",
  "start_date": "2026-01-26",
  "current_day": 32,
  "timeline_generated": true
}
```

## 메시지 설계 원칙

GPT 건강 상담 대화에서 추출한 패턴.
사용자가 "왜 안 빠지지?"를 검색하기 **전에** 앱이 먼저 말해주는 구조.

### 안심 패턴 (모든 메시지의 뼈대)

하나의 타임라인 이벤트는 단순 문장이 아니라, **3단 구조**를 가진다:

| 단계 | 역할 | 예시 (금주 D+28) |
|---|---|---|
| **fact** | 지금 몸에서 일어나고 있는 사실 | "체수분 균형이 재조정되고, 간이 글리코겐 저장 능력을 회복하는 중입니다" |
| **feeling** | 그래서 사용자가 느낄 수 있는 감정/의문 | "체중이 안 빠진다고 느끼실 수 있습니다. 이건 체성분이 재배치되는 구간이라 정상입니다" |
| **action** | 지금 할 수 있는 구체적 행동 | "체중 대신 허리둘레를 재보세요. 아침 기상 직후, 배꼽 기준으로" |

> **핵심: "fact → feeling → action" 순서.**
> 사실을 먼저 알려주고 → 감정을 인정하고 → 대안 행동을 제시한다.
> 응원("화이팅!", "잘하고 있어요!")이 아니라 **근거 기반 안심**.

### 선제적 메시지의 타이밍

사용자가 불안해지는 시점은 예측 가능하다.

| 예상 심리 | 타이밍 | 앱이 먼저 해야 할 말 |
|---|---|---|
| "금단 증상 힘들다" | D+3~7 | 금단의 원인 + 피크 시점 + "여기만 넘기면" |
| "효과 있는 거 맞아?" | D+14~21 | 눈에 안 보이지만 진행 중인 변화 목록 |
| "한 달이나 했는데 왜 똑같지?" | D+28~35 | **체중 정체의 의학적 원인** + 대안 지표 |
| "계속할 이유가 있나?" | D+42~56 | 여기서부터 보이기 시작하는 변화 + 체크 방법 |

> D+28~35가 이탈 위험이 가장 높은 구간.
> 이 구간의 메시지가 가장 풍성해야 한다.

## 타임라인

### 구조: 2계층 — 단계(Stage) + 이벤트(Event)

금욕 등록이 완료되면, 해당 종류 + 개인화 변수를 기반으로 타임라인이 생성된다.
타임라인은 **단계(Stage)** 와 **이벤트(Event)** 2계층으로 구성된다.

```
타임라인 = 단계 목록

단계 (Stage):
  - stage: 단계 번호 (1, 2, 3...)
  - name: 단계 이름 ("해독기", "초기 회복기" 등)
  - day_from: 시작일 (D+N)
  - day_to: 종료일 (D+N)
  - summary: 이 구간에서 일어나는 일 한 줄 요약
  - events: 이벤트 목록

이벤트 (Event):
  - day: D+N (정수)
  - fact: 지금 몸/뇌에서 일어나고 있는 사실
  - feeling: 사용자가 느낄 수 있는 감정/의문 + 안심 (nullable)
  - action: 지금 할 수 있는 구체적 행동 (nullable)
  - is_proactive: 선제적 알림 여부 (불안 예상 시점)
  - condition: 개인화 조건 (nullable)
```

> **단계(Stage)**: "나는 지금 5단계 중 3단계에 있고, 10일 후에 4단계에 접어든다"
> **이벤트(Event)**: "이 단계에서 구체적으로 무슨 일이 일어나고 있는지"
>
> 단계는 큰 그림을 보여주고, 이벤트는 디테일을 채운다.

### 프리셋 타임라인 데이터

타임라인 원본 데이터는 **서버 측 JSON 파일**로 관리한다.

```
backend/
  data/
    timelines/
      alcohol.json    # 금주 타임라인
      smoking.json    # 금연 타임라인
      diet.json       # 식단 타임라인
```

### 개인화 전략: AI 1회 생성 + 주간 체크인 재조정

타임라인 날짜(D+N)는 사람마다 다르다. 나이, 체형, 생활 습관, 운동 여부 등에 따라
회복 속도가 달라지기 때문. 이를 반영하기 위해 **AI 기반 개인화**를 사용한다.

#### 전략 요약

| 시점 | AI 호출 | 용도 | 비용 |
|---|---|---|---|
| 금욕 등록 시 | 1회 | 사용자 변수 기반으로 기본 템플릿의 날짜/메시지 조정 | ~$0.05 |
| 주간 체크인 시 | 주 1회 | 생활 변화 반영하여 남은 타임라인 재조정 | ~$0.05 |
| 대시보드/타임라인 조회 시 | 없음 | DB에서 읽기만 | $0 |

> 월 비용: ~$0.25/유저. 토이 프로젝트 수준에서 부담 없음.

#### 등록 시 1회 생성 흐름

```
[금욕 등록]
    │  사용자 입력: 종류, 개인 변수 (나이, 체중, 음주력 등)
    │
    ▼
[AI 호출]
    │  입력: 기본 템플릿(alcohol.json 등) + 사용자 변수
    │  출력: 조정된 타임라인 JSON
    │    - D+7 → D+9 (나이 40+이라 회복 느림)
    │    - D+28 fact에 "운동 중이므로 글리코겐 영향" 추가
    │    - D+35 테스토스테론 이벤트 포함 (남성)
    │    - 음주력 10년+ → 간 손상 관련 메시지 강화
    │
    ▼
[조정된 타임라인을 DB에 저장]
    │
    ▼
[이후 조회는 DB 읽기만 → AI 호출 없음]
```

#### 주간 체크인

사용자의 생활 패턴은 계속 변한다 (운동 시작, 야근 폭주, 수면 불규칙 등).
등록 시점의 스냅샷 하나로는 정확도가 떨어지므로, **주간 체크인**으로 보정한다.

##### 체크인 항목

| 질문 | 응답 타입 | 용도 |
|---|---|---|
| "이번 주 운동 했나요?" | 예/아니오 | 대사 속도 반영 |
| "수면은 어땠나요?" | 좋음/보통/나쁨 | 회복 속도 반영 |
| "식사는 규칙적이었나요?" | 예/아니오 | 체성분 변화 반영 |
| "충동이 있었나요?" | 예/아니오 | 금단 진행도 반영 |

> 금욕 종류별로 질문이 달라질 수 있다.
> 예: 금연 → "대체 행동(간식 등)이 늘었나요?", 식단 → "외식이 몇 회였나요?"

##### 체크인 플로우

```
[주 1회 앱 진입 시 체크인 카드 표시]
    │
    ├─ 4개 질문 (탭 형태, 30초 이내 완료)
    │
    ▼
[AI 1회 호출]
    │  입력: 현재 저장된 타임라인 + 체크인 응답 + 사용자 변수
    │  출력: 남은 이벤트들의 날짜/메시지 재조정
    │    - 운동 시작 → 회복 속도 앞당김
    │    - 수면 나쁨 → 회복 둔화 반영
    │    - 충동 있음 → 격려 메시지 강화 시점 조정
    │
    ▼
[DB에 남은 타임라인 업데이트]
```

> 체크인은 강제가 아니다. 안 하면 기존 타임라인 유지.
> 체크인 자체가 앱 재방문 동기가 됨 (리텐션 효과).

#### condition 필드 (기본 템플릿 필터링)

AI 호출 이전에, 기본 템플릿 단계에서도 condition 기반 필터링이 적용된다.

| 변수 | condition 예시 | 효과 |
|---|---|---|
| 성별 | `{ "gender": "male" }` | 남성 전용 이벤트 포함/제외 |
| 운동 여부 | `{ "has_exercise": true }` | 운동 중 체중 정체 설명 분기 |
| 음주/흡연 기간 | `{ "drinking_years_gte": 3 }` | 장기 손상 관련 메시지 포함 |

> condition 필터링은 룰 기반(코드). AI가 아닌 서버 로직으로 처리.
> AI는 condition 필터링 후의 결과물에 대해 날짜/메시지를 조정하는 역할.

## 네비게이션 구조

### 하단 탭 바 (3탭)

```
┌──────────────────────────────────┐
│                                  │
│         [ 화면 콘텐츠 ]            │
│                                  │
├──────────────────────────────────┤
│   🏠 홈    📈 타임라인    👤 마이   │
└──────────────────────────────────┘
```

| 탭 | 아이콘 | 대상 화면 | 설명 |
|---|---|---|---|
| 홈 | 🏠 | `/dashboard` | 오늘의 메시지 + 금욕 카드 + 체크인 카드 |
| 타임라인 | 📈 | `/timeline` | 내 금욕 목록 → 타임라인 상세 |
| 마이 | 👤 | `/mypage` | 프로필, 설정, 로그아웃 |

### 화면 흐름

```
[홈 탭]
    │
    ├─ 금욕 카드 탭 ──────→ [타임라인 상세] (타임라인 탭으로 이동)
    ├─ "금욕 추가" ────────→ [금욕 등록] (모달 or 풀스크린)
    └─ 체크인 카드 ────────→ [주간 체크인] (풀스크린)

[타임라인 탭]
    │
    ├─ 금욕이 1개 ─────────→ 바로 타임라인 상세 표시
    └─ 금욕이 2개+ ────────→ 금욕 선택 리스트 → 타임라인 상세

[마이 탭]
    │
    ├─ 닉네임, 이메일 표시
    ├─ 내 금욕 관리 (삭제 등)
    └─ 로그아웃
```

### 상단 헤더

탭 바와 별개로, 각 화면 상단에 컨텍스트 헤더를 표시한다.

```
홈:           "restrainter"  (앱 이름)
타임라인 목록: "타임라인"
타임라인 상세: "← 금주 타임라인       D+32"
금욕 등록:    "← 금욕 시작하기"
주간 체크인:  "← 주간 체크인"
마이:         "마이페이지"
```

> `←` 는 뒤로가기 버튼. 탭 직속 화면에는 없고, push된 화면에만 표시.
